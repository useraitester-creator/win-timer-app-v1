# Дизайн: передача затраченного времени в Битрикс24

Дата: 2026-06-11
Статус: на ревью

## Цель

Передавать затраченное время по задачам трекера, связанным с Битрикс24, в портал.
Выгрузка идёт из окна «История» по выбранным галочками интервалам: выбранные
(ещё не переданные) интервалы **суммируются в одну запись**. В таблице интервалов
появляется колонка с отметкой о передаче и сохранённым id записи.

## Куда пишем время

Зависит от типа связи задачи (`task.bitrix = {"source", "id"}`):

- **Проект** (`source = "project"`) → новый элемент СПА **1092** (журнал работ):
  - `entityTypeId = 1092`
  - `parentId150` = id проекта (СПА 150)
  - `assignedById` = текущий пользователь (из `profile`)
  - `ufCrm88HoursWork` = сумма часов (double)
  - `ufCrm88CommentWork` = название (обязательно)
  - `ufCrm88DateWork` = дата (см. умолчания)
- **Задача** (`source = "task"`) → `task.elapseditem.add`:
  - `TASKID` = id задачи
  - `ARFIELDS = {"SECONDS": сумма_секунд, "COMMENT_TEXT": название}`

Если задача не связана с Битрикс — передача недоступна.

## Модель данных

`win_timer_app/models.py` — `Session`:
- Добавить `bitrix_record_id: str | None = None` — id записи в Битрикс, в которую
  интервал был передан. `None` = не передан. Сериализуется в `to_dict`/`from_dict`.

«Отметка о передаче» = `bitrix_record_id is not None`.

## Поток передачи (окно «История»)

1. Пользователь отмечает интервалы чекбоксами (есть «Выделить всё»).
2. Жмёт **«Передать в Битрикс»**.
3. Из отмеченных берём только **не переданные** (`bitrix_record_id is None`).
   Если таких нет — сообщение, выход.
4. Запрашиваем **название** (`QInputDialog`, предзаполнено названием задачи).
   Пустое название → отмена.
5. Суммируем длительности выбранных интервалов.
6. В фоне (`QThread`):
   - проект → `client.add_project_time(project_id, date, hours, comment, user_id)`
     (→ `crm.item.add`, entityTypeId 1092) → id записи;
   - задача → `client.add_task_time(task_id, seconds, comment)`
     (→ `task.elapseditem.add`) → id записи.
7. Успех: каждому переданному интервалу проставляем `bitrix_record_id = id`,
   сохраняем, перечитываем таблицу. Ошибка: предупреждение (токен экранирован).

Кнопка «Передать в Битрикс» активна, только если задача связана с Битрикс и есть
хотя бы один непереданный интервал.

## Клиент (`win_timer_app/bitrix.py`)

- `add_project_time(project_id, date_iso, hours, comment, responsible_id) -> str`
  — `crm.item.add` (entityTypeId 1092), возвращает id элемента (`result["item"]["id"]`).
- `add_task_time(task_id, seconds, comment) -> str`
  — `task.elapseditem.add` с порядком параметров `TASKID`, затем `ARFIELDS`.
- Константа `WORKLOG_ENTITY_TYPE_ID = 1092`.

## Контроллер

- `mark_sessions_transferred(task_id, session_ids, record_id)` — проставляет
  `bitrix_record_id` указанным сессиям и сохраняет.
- Хелпер выбора непереданных сессий (или это делает диалог).

## UI (`SessionEditDialog`)

- В таблицу добавляется колонка **«Передано»**: для переданных интервалов
  показывает id записи (для непереданных — пусто).
- Кнопка **«Передать в Битрикс»** рядом с «Удалить запись». Включена при наличии
  связи с Битрикс и непереданных отмеченных интервалов.
- Передача — в фоне; на время — индикация недоступности кнопки.

## Умолчания

- `ufCrm88DateWork` (1092) = **дата последнего** из выбранных интервалов.
- Часы для 1092 = сумма_секунд / 3600, округление до **2 знаков**.
- `task.elapseditem` фиксирует дату сам (текущая) — отдельная дата не нужна.

## Идемпотентность и редактирование

- Переданные интервалы (`bitrix_record_id` задан) исключаются из новой передачи —
  двойного учёта нет.
- Изменение/удаление уже переданного интервала в трекере **не** трогает запись в
  Битрикс (вне рамок этого шага; при необходимости — отдельно).

## Тестирование

TDD (юнит, на фейках):
- клиент: `add_project_time` формирует `crm.item.add` с нужными полями (1092,
  parentId150, hours, comment, date, assignedById); `add_task_time` формирует
  `task.elapseditem.add` с порядком `TASKID`/`ARFIELDS`; оба возвращают id.
- модель: сериализация `bitrix_record_id`.
- контроллер: `mark_sessions_transferred` проставляет id и сохраняет.

Проверка на реальном API:
- `task.elapseditem.add` — на тестовой задаче 50032: добавить, проверить, удалить
  (`task.elapseditem.delete`), чтобы не оставлять мусор.
- 1092 (`crm.item.add`) — формат проверить аккуратно; реальную запись создавать
  только с явного согласия (иначе остаётся запись в CRM).

UI — проверка запуском (offscreen-драйвер + скриншоты): колонка «Передано»,
кнопка, запрос названия, пометка интервалов.

## Вне рамок

- Обновление/удаление записи в Битрикс при правке/удалении интервала.
- Передача времени по проектам в учёт времени задач и наоборот.
- Авто-передача (только ручная по галочкам).
